## Image Generation Platforms

### Supported Platforms
Defined in `foundation/lib/constants.ts`:
```typescript
export const GENERATION_PLATFORMS = ['gpt', 'jimeng'] as const
export type GenerationPlatform = 'gpt' | 'jimeng'
```

### Data Lineage

All image generation follows a consistent traceability chain:

```
ModelImage (real generated by platform)
  ↓
StyleImage (stub record: modelImageId + pose description)
  ↓
FirstFrame (real generated: styleImageId + sceneId + composition + generationPath)
```

**generationPath** field in `FirstFrame` table identifies the generation platform:
- `'gpt'` - GPT platform
- `'jimeng'` - Jimeng platform

Unique key: `[styleImageId, sceneId, composition, generationPath]`

### Jimeng Flow Details

Jimeng image generation directly calls the API with `modelImageId, poseId, sceneId`, but database records still follow the full GPT-style chain for traceability:

1. Create stub `styleImage` record (hash of modelImageId + pose description)
2. Create `firstFrame` record with `generationPath = 'jimeng'`
